LV_EX_CalcViewSize(HLV[, Rows])

Calculates the approximate width and height required to display a given number of items.

Parameters:
    HLV            -  HWND of the list-view.
    Optional -------
    Rows           -  Number of visible items to be displayed in the control.
                      Default = 0 (total number of items)
Return values:
    On success:    Object containing the following keys:
                   W : Width in pixels.
                   H : Height in pixels.
    On failure:    False.
Remarks:
    The returned values specify the width and height of the client area including the header and the
    scrollbars, if any.
MSDN:
    http://msdn.microsoft.com/en-us/library/bb774883(VS.85).aspx

LV_EX_FindString(HLV, Str[, Start[, Partial]])

Searches the first column for an item containing the specified string.

Parameters:
    HLV            -  HWND of the list-view.
    Str            -  String to search for.
                      Default: 0

    Start          -  The 1-based index of the row to begin the search with

Return values:
    On success:    1-based row index of the matching row.

LV_EX_GetItemRect(HLV[, Row[, LVIR]])

Retrieves the bounding rectangle for all or part of an item in the current view.

Parameters:
    HLV            -  HWND of the list-view.
    -------------- Optional:
    Row            -  1-based index of the list-view row.
                      Default: 1
    LVIR           -  Portion of the list-view item from which to retrieve the bounding rectangle.
                      This can be one of the following values:
                      0 (LVIR_BOUNDS)
                         Returns the bounding rectangle of the entire item, including the icon and label.
                      1 (LVIR_ICON)
                         Returns the bounding rectangle of the icon or small icon.
                      2 (LVIR_LABEL)
                         Returns the bounding rectangle of the item text.
                      3 (LVIR_SELECTBOUNDS)
                         Returns the union of the LVIR_ICON and LVIR_LABEL rectangles, but excludes 
                         columns in report view.
                      Default: 0
Return values:
    On success:    Object containing the following keys:
                   X : X-position of the rectangle's left side (left)
                   Y : Y-position of the rectangle's top side (top)
                   R : X-position of the rectangle's right side (right)
                   B : Y-position of the rectangle's bottom side (bottom)
                   W : Width
                   H : Height
    On failure:    False.
MSDN:
    http://msdn.microsoft.com/en-us/library/bb761049(VS.85).aspx

LV_EX_SetBkImage(HLV, ImagePath[, Width[, Height]])

Sets the background image in a list-view control.

Parameters:
    HLV            -  HWND of the list-view.
    ImagePath      -  File path of the bitmap.
                      Pass an empty string ("") to remove the background image.
    Optional -------
    Width          -  Width to show the image in pixels.
                      Default: "" (width of the list-view's client area)
    Height         -  Height to show the image in pixels.
                      Default: "" (height of the list-view's client area)
Return values:
    On success:    Nonzero.
    On failure:    False.
Remarks:
    The extended style LVS_EX_DOUBLEBUFFER is set internally to avoid drawing issues.
    On Win XP you have to set -LV0x20 in the list-view's options. 
    Otherwise the image won't be visible within the first column.
MSDN:
    http://msdn.microsoft.com/en-us/library/bb761155(VS.85).aspx
